Coffee machine

Implement a system to manage a coffee preparing and selling machine (using Java Collections). The system must meet the following requirements.

R1: Coffee

The coffee machine can distribute several types of coffees. Each coffee is characterized by its unique name selling price and products amount used for preparation coffee, milk, water For instance:

Method addCoffee() of class CoffeeMachine let the administrator to add the description of a new coffee. To obtain the price of a coffee, the method getPrice() is provided. If a price is requested for a coffee name that does not exist, the method returns an error code (e.g., -1.0).

R2: Payment cards

The coffee machine works by means of payment cards. Each card is characterized by a unique ID (integer number) and the available credit. For instance:

Card ID
 Credit
7 50000

Cards are recharged through the method rechargeCard() of class CoffeeMachine. If a non-existing ID is provided a new card is created, otherwise the existing card is recharged. Further, it is possible to query the available credit of a given card with method getCredit(). If a non-existing card ID is queried, the method returns an error code (e.g., -1.0).

R3: Machine refill

The Coffee machine consist of products for preparing coffee (coffee, milk, water). Administrator can refill products based on name of product and amount. To this aim method refillProduct() is provided.
Given the name of a coffee (e.g., "Cappuccino") it is possible to obtain the total number of available products to prepare current coffee by means of method availableCoffee(). The method counts all products to prepare coffee and return boolean value with result.

R4: Selling

The coffee machine sells a product by specifying the coffee name and the payment card ID in method sell(). The method returns the column where the coffee was picked from. If codes are invalid (either coffee or card), the coffee ran out, or credit on payment card is not enough to buy the requested item, an error code is returned (e.g., -1). On normal behavior, the machine decrements the credit on payment card and the quantity of available coffee products for the selected column.
